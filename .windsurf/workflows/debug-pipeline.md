---
description: 排查主线 pipeline 错误的标准步骤
---

1. 用 `code_search` 定位报错涉及的代码路径（不要先用 grep）
2. 如果涉及第三方库行为，用 MCP `context7` 查询官方文档
3. 如果问题仍复杂，用 MCP `sequential-thinking` 做分步推理：
   - 先拆可能原因
   - 给排查顺序
   - 每步说明验证目标
   - 假设不成立时修正推理路径
4. 确认 root cause 后，最小化修复（不得跨模块联动改动）
5. 修复只针对当前授权模块；下游受影响的问题单独立卡
6. 修复后重跑受影响的 pytest，确认无回归
